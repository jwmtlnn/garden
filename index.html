This website is for publishing my research project while it happens. During the research i build a body of texts in a format of hypertext, with a simple markup language (fin. merkintÃ¤kieli) called Markdown. What that means, is that these texts link into each other constituting a non-linear repository where the contents relate to each other. For a taste of this you can jump straight into the Actor-Network Theory note, my Research Plan or see examples of similar (or way fancier) implementations in the Digital Garden note.

All of this is managed in a personal knowledge base system called Obsidian. This repository in Obsidian is called a Vault. When it's published, it can be described with terms such as Digital Garden or Wiki (without the possibility for reader editing). This site has been translated straight from my Obsidian Vault into a www-publishable HTML with Nathan George's HTML Export Plugin. It is then hosted in and published with GitHub and it's Pages functionality.

Info

This is very much a work in progress. The maturity of the content lives and that's kinda the whole point of it. Not a single note is permanent, fixed or "ready". Also not everything in my Vault will be published right away. This site itself is a static epoch of my Vault and has to be manually updated into the web whenever wanted to.

Caution

The functionalities of this website are not perfect. Translating from Obsidian flavoured Markdown to HTML (with all the third-party plugins that i use) does not mirror my Vault perfectly. I focus on content, but will gladly hear of any hiccups in the user interface. Main problems are:

  • Citations using Pandoc citation keys have their technical restrictions and might not be presented in the most conventional or human-readable format.
  • Site's layout has it's quirks, for example mobile device accessibility is not good and listings may have unnecessary line breaks.
  • Schedules written in Markwhen do not work in web view.
