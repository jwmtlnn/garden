Welcome! 🌱

This website is for publishing my research project while it happens. During the research i build a body of texts in a format of hypertext, with a language called Markdown. What that means, is that these texts link into each other constituting a non-linear repository where the contents relate to each other. For a taste of this you can jump straight into my Research Plan or see examples of similar (way fancier) implementations in the Digital Garden text.

All of this is managed in a personal knowledge base system called Obsidian. This repository in Obsidian is called a Vault. When it's published, it can be described with terms such as Digital Garden, Pattern Language or even Wiki. This site has been translated straight from my Obsidian Vault into a WWW-publishable HTML with Nathan George's HTML Export Plugin. It is then hosted in and published with GitHub and it's Pages functionality.

Info

This is very much a work in progress, and not everything that i produce will be published. This site itself is a static epoch of my Vault and has to be updated into the web whenever needed.

The project has been started: 1.1.2025
Last web update: 8.3.2025

Caution

The functionalities of this website are not perfect. Due to technical restrictions of the plugin when translating from Obsidian flavoured Markdown to HTML (with all the third-party plugins that i use), it does not mirror my Vault perfectly. Main problems being:

  • Citations are presented as Pandoc format citation keys instead of human-readable citations
    • f.e. @latourScienceActionHow1987 and not (Latour 1987)
  • Site's layout has it's quirks
    • f.e. some icons are bolded or inappropriately scaled and listings may have unnecessary line breaks
  • The schedule written in Markwhen does not represent itself in browser as intended, and instead downloads on your computer as raw Markwhen syntax.
